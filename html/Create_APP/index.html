Criar um aplicativo

Se você chegou até aqui, é provável que tenha instalado o Streamlit e executado o básico em nosso guia de conceitos principais . Se não, agora é um bom momento para dar uma olhada.

A maneira mais fácil de aprender a usar o Streamlit é experimentar você mesmo. Enquanto você lê este guia, teste cada método. Enquanto seu aplicativo estiver em execução, toda vez que você adicionar um novo elemento ao seu script e salvar, a interface do usuário do Streamlit perguntará se você deseja executar novamente o aplicativo e visualizar as alterações. Isso permite que você trabalhe em um loop interativo rápido: você escreve algum código, salva, revisa a saída, escreve um pouco mais e assim por diante, até ficar satisfeito com os resultados. O objetivo é usar o Streamlit para criar um aplicativo interativo para seus dados ou modelo e, ao longo do caminho, usar o Streamlit para revisar, depurar, aperfeiçoar e compartilhar seu código.

Neste guia, você usará os principais recursos do Streamlit para criar um aplicativo interativo; explorando um conjunto de dados público do Uber para embarques e desembarques na cidade de Nova York. Quando terminar, você saberá como buscar e armazenar dados em cache, desenhar gráficos, plotar informações em um mapa e usar widgets interativos, como um controle deslizante, para filtrar resultados.

Crie seu primeiro aplicativo

O Streamlit é mais do que apenas uma maneira de criar aplicativos de dados, é também uma comunidade de criadores que compartilham seus aplicativos e ideias e ajudam uns aos outros a melhorar seu trabalho. Por favor, junte-se a nós no fórum da comunidade. Adoramos ouvir suas perguntas, ideias e ajudá-lo a resolver seus bugs - venha hoje mesmo!

  1. O primeiro passo é criar um novo script Python. Vamos chamá-lo uber_pickups.py .

  2. Abra uber_pickups.py em seu IDE ou editor de texto favorito e adicione estas linhas: 

    import streamlit as st
import pandas as pd
import numpy as np

  3. Todo bom aplicativo tem um título, então vamos adicionar um: 

    st.title('Uber pickups in NYC')

  4. Agora é hora de executar o Streamlit na linha de comando: 

    streamlit run uber_pickups.py

    A execução de um aplicativo Streamlit não é diferente de qualquer outro script Python. Sempre que precisar visualizar o aplicativo, você pode usar este comando.

Buscar alguns dados

Agora que você tem um aplicativo, a próxima coisa que você precisa fazer é buscar o conjunto de dados do Uber para embarques e desembarques na cidade de Nova York.

  1. Vamos começar escrevendo uma função para carregar os dados. Adicione este código ao seu script: 

    DATE_COLUMN = 'date/time'
DATA_URL = ('https://s3-us-west-2.amazonaws.com/'
         'streamlit-demo-data/uber-raw-data-sep14.csv.gz')

def load_data(nrows):
    data = pd.read_csv(DATA_URL, nrows=nrows)
    lowercase = lambda x: str(x).lower()
    data.rename(lowercase, axis='columns', inplace=True)
    data[DATE_COLUMN] = pd.to_datetime(data[DATE_COLUMN])
    return data

    Você notará que load_data é uma função simples e antiga que baixa alguns dados, os coloca em um dataframe do Pandas e converte a coluna de data de texto para data e hora. A função aceita um único parâmetro ( nrows ), que especifica o número de linhas que você deseja carregar no dataframe.

  2. Agora vamos testar a função e revisar a saída. Abaixo da sua função, adicione estas linhas: 

    # Create a text element and let the reader know the data is loading.
data_load_state = st.text('Loading data...')
# Load 10,000 rows of data into the dataframe.
data = load_data(10000)
# Notify the reader that the data was successfully loaded.
data_load_state.text('Loading data...done!')

    Você verá alguns botões no canto superior direito do seu aplicativo perguntando se deseja executá-lo novamente. Escolha Sempre reexecutar e você verá suas alterações automaticamente sempre que salvar.

Ok, isso é desanimador...

Acontece que leva muito tempo para baixar dados e carregar 10.000 linhas em um dataframe. Converter a coluna de data em data e hora também não é um trabalho rápido. Você não deseja recarregar os dados toda vez que o aplicativo for atualizado - felizmente, o Streamlit permite armazenar os dados em cache.

Cache sem esforço

  1. Tente adicionar @st.cache antes da load_data declaração: 

    @st.cache
def load_data(nrows):

  2. Em seguida, salve o script e o Streamlit executará novamente seu aplicativo automaticamente. Como esta é a primeira vez que você está executando o script com @st.cache , você não verá nada mudar. Vamos ajustar seu arquivo um pouco mais para que você possa ver o poder do cache.

  3. Substitua a linha data_load_state.text('Loading data...done!') por esta: 

    data_load_state.text("Done! (using st.cache)")

  4. Agora salve. Veja como a linha que você adicionou apareceu imediatamente? Se você der um passo para trás por um segundo, isso é realmente incrível. Algo mágico está acontecendo nos bastidores, e é preciso apenas uma linha de código para ativá-lo.

Como funciona?

Vamos tomar alguns minutos para discutir como @st.cache realmente funciona.

Quando você marca uma função com a anotação de cache do Streamlit, ele informa ao Streamlit que sempre que a função é chamada, ela deve verificar três coisas:

  1. O bytecode real que compõe o corpo da função
  2. Código, variáveis ​​e arquivos dos quais a função depende
  3. Os parâmetros de entrada com os quais você chamou a função

Se esta é a primeira vez que o Streamlit vê esses itens, com esses valores exatos e nessa combinação exata, ele executa a função e armazena o resultado em um cache local. Na próxima vez que a função for chamada, se os três valores não tiverem sido alterados, o Streamlit saberá que pode pular a execução da função completamente. Em vez disso, ele lê a saída do cache local e a passa para o chamador - como mágica.

"Mas, espere um segundo", você está dizendo para si mesmo, "isso parece bom demais para ser verdade. Quais são as limitações de todo esse molho incrível?"

Bem, existem alguns:

  1. O Streamlit verificará apenas as alterações no diretório de trabalho atual. Se você atualizar uma biblioteca Python, o cache do Streamlit só notará isso se essa biblioteca estiver instalada dentro do seu diretório de trabalho.
  2. Se sua função não é determinística (ou seja, sua saída depende de números aleatórios), ou se ela extrai dados de uma fonte variável no tempo externa (por exemplo, um serviço de cotações do mercado de ações ao vivo), o valor em cache não será nenhum. Mais sábio.
  3. Por fim, você não deve alterar a saída de uma função em cache, pois os valores em cache são armazenados por referência (por motivos de desempenho e para poder oferecer suporte a bibliotecas como o TensorFlow). Observe que, aqui, o Streamlit é inteligente o suficiente para detectar essas mutações e mostrar um aviso alto explicando como corrigir o problema.

Embora seja importante ter em mente essas limitações, elas tendem a não ser um problema surpreendente na maioria das vezes. Naquela época, esse cache é realmente transformador.

Inspecione os dados brutos

É sempre uma boa ideia dar uma olhada nos dados brutos com os quais você está trabalhando antes de começar a trabalhar com eles. Vamos adicionar um subcabeçalho e uma impressão dos dados brutos ao aplicativo: 

st.subheader('Raw data')
st.write(data)

No guia de conceitos principais, você aprendeu que st.write renderizará quase tudo que você passar para ele. Nesse caso, você está passando um dataframe e está renderizando como uma tabela interativa.

st.write tenta fazer a coisa certa com base no tipo de dados da entrada. Se não estiver fazendo o que você espera, você pode usar um comando especializado como st.dataframe . Para obter uma lista completa, consulte a referência da API .

Desenhe um histograma

Agora que você teve a chance de dar uma olhada no conjunto de dados e observar o que está disponível, vamos dar um passo adiante e desenhar um histograma para ver quais são os horários de maior movimento do Uber na cidade de Nova York.

  1. Para começar, vamos adicionar um subtítulo logo abaixo da seção de dados brutos: 

    st.subheader('Number of pickups by hour')

  2. Use o NumPy para gerar um histograma que divide os tempos de coleta agrupados por hora: 

    hist_values = np.histogram(
    data[DATE_COLUMN].dt.hour, bins=24, range=(0,24))[0]

  3. Agora, vamos usar o método do Streamlit st.bar_chart() para desenhar este histograma. 

    st.bar_chart(hist_values)

  4. Salve seu roteiro. Esse histograma deve aparecer em seu aplicativo imediatamente. Após uma rápida revisão, parece que o horário mais movimentado é 17:00 (17:00).

Para desenhar este diagrama usamos o bar_chart() método nativo do Streamlit, mas é importante saber que o Streamlit suporta bibliotecas de gráficos mais complexas como Altair, Bokeh, Plotly, Matplotlib e muito mais. Para obter uma lista completa, consulte bibliotecas de gráficos compatíveis .

Plotar dados em um mapa

Usar um histograma com o conjunto de dados do Uber nos ajudou a determinar quais são os horários mais movimentados para coletas, mas e se quiséssemos descobrir onde as coletas estavam concentradas em toda a cidade. Embora você possa usar um gráfico de barras para mostrar esses dados, não seria fácil interpretá-los, a menos que estivesse intimamente familiarizado com as coordenadas latitudinais e longitudinais da cidade. Para mostrar a concentração de coleta, vamos usar a função Streamlit st.map() para sobrepor os dados em um mapa da cidade de Nova York.

  1. Adicione um subtítulo para a seção: 

    st.subheader('Map of all pickups')

  2. Use a st.map() função para plotar os dados: 

    st.map(data)

  3. Salve seu roteiro. O mapa é totalmente interativo. Experimente fazendo uma panorâmica ou ampliando um pouco.

Depois de desenhar seu histograma, você determinou que o horário de maior movimento para as coletas do Uber era 17:00. Vamos redesenhar o mapa para mostrar a concentração de coletas às 17:00.

  1. Localize o seguinte trecho de código: 

    st.subheader('Map of all pickups')
st.map(data)

  2. Substitua-o por: 

    hour_to_filter = 17
filtered_data = data[data[DATE_COLUMN].dt.hour == hour_to_filter]
st.subheader(f'Map of all pickups at {hour_to_filter}:00')
st.map(filtered_data)

  3. Você deve ver a atualização de dados instantaneamente.

Para desenhar este mapa, usamos a st.map função incorporada ao Streamlit, mas se você quiser visualizar dados de mapas complexos, recomendamos que você dê uma olhada no arquivo st.pydeck_chart .

Filtrar resultados com um controle deslizante

Na última seção, quando você desenhou o mapa, o tempo usado para filtrar os resultados foi codificado no script, mas e se quisermos permitir que um leitor filtre os dados dinamicamente em tempo real? Usando os widgets do Streamlit você pode. Vamos adicionar um controle deslizante ao aplicativo com o st.slider() método.

  1. Localize hour_to_filter e substitua-o por este snippet de código: 

    hour_to_filter = st.slider('hour', 0, 23, 17)  # min: 0h, max: 23h, default: 17h

  2. Use o controle deslizante e veja a atualização do mapa em tempo real.

Use um botão para alternar dados

Os controles deslizantes são apenas uma maneira de alterar dinamicamente a composição do seu aplicativo. Vamos usar a st.checkbox função para adicionar uma caixa de seleção ao seu aplicativo. Usaremos essa caixa de seleção para mostrar/ocultar a tabela de dados brutos na parte superior do seu aplicativo.

  1. Localize estas linhas:

st.write(data)

  2. Substitua essas linhas pelo seguinte código: 

    if st.checkbox('Show raw data'):
    st.subheader('Raw data')
    st.write(data)

Vamos juntar tudo

É isso, você chegou até o fim. Aqui está o script completo para nosso aplicativo interativo.

Estrela

Gorjeta

Se você pulou adiante, depois de criar seu script, o comando para executar o Streamlit é streamlit run [app name] . 

import streamlit as st
import pandas as pd
import numpy as np

st.title('Uber pickups in NYC')

DATE_COLUMN = 'date/time'
DATA_URL = ('https://s3-us-west-2.amazonaws.com/'
            'streamlit-demo-data/uber-raw-data-sep14.csv.gz')

@st.cache
def load_data(nrows):
    data = pd.read_csv(DATA_URL, nrows=nrows)
    lowercase = lambda x: str(x).lower()
    data.rename(lowercase, axis='columns', inplace=True)
    data[DATE_COLUMN] = pd.to_datetime(data[DATE_COLUMN])
    return data

data_load_state = st.text('Loading data...')
data = load_data(10000)
data_load_state.text("Done! (using st.cache)")

if st.checkbox('Show raw data'):
    st.subheader('Raw data')
    st.write(data)

st.subheader('Number of pickups by hour')
hist_values = np.histogram(data[DATE_COLUMN].dt.hour, bins=24, range=(0,24))[0]
st.bar_chart(hist_values)

# Some number in the range 0-23
hour_to_filter = st.slider('hour', 0, 23, 17)
filtered_data = data[data[DATE_COLUMN].dt.hour == hour_to_filter]

st.subheader('Map of all pickups at %s:00' % hour_to_filter)
st.map(filtered_data)

Compartilhe seu aplicativo

Depois de criar um aplicativo Streamlit, é hora de compartilhá-lo! Para mostrá-lo ao mundo, você pode usar o Streamlit Cloud para implantar, gerenciar e compartilhar seu aplicativo gratuitamente.

Funciona em 3 passos simples:

  1. Coloque seu aplicativo em um repositório público do GitHub (e certifique-se de que ele tenha um requirements.txt!)
  2. Faça login em share.streamlit.io
  3. Clique em 'Implantar um aplicativo' e cole no URL do GitHub
