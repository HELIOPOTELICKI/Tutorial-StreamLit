Principais conceitos

Trabalhar com Streamlit é simples. Primeiro, você espalha alguns comandos Streamlit em um script Python normal, depois o executa com streamlit run : 

streamlit run your_script.py [-- script args]

Assim que você executar o script conforme mostrado acima, um servidor Streamlit local será ativado e seu aplicativo será aberto em uma nova guia no navegador da Web padrão. O aplicativo é sua tela, onde você desenhará gráficos, texto, widgets, tabelas e muito mais.

O que é desenhado no aplicativo é com você. Por exemplo st.text , grava texto bruto em seu aplicativo e st.line_chart desenha - você adivinhou - um gráfico de linhas. Consulte nossa documentação da API para ver todos os comandos disponíveis para você.

Observação

Ao passar ao seu script alguns argumentos personalizados, eles devem ser passados ​​após dois traços. Caso contrário, os argumentos serão interpretados como argumentos para o próprio Streamlit.

Outra maneira de executar o Streamlit é executá-lo como um módulo Python. Isso pode ser útil ao configurar um IDE como o PyCharm para trabalhar com o Streamlit: 

# Running
$ python -m streamlit run your_script.py

# is equivalent to:
$ streamlit run your_script.py

Você também pode passar um URL para streamlit run ! Isso é ótimo quando combinado com Github Gists. Por exemplo: 

$ streamlit run https://raw.githubusercontent.com/streamlit/demo-uber-nyc-pickups/master/streamlit_app.py

Fluxo de desenvolvimento

Toda vez que você quiser atualizar seu aplicativo, salve o arquivo de origem. Quando você faz isso, o Streamlit detecta se há uma alteração e pergunta se você deseja executar novamente seu aplicativo. Escolha "Sempre executar novamente" no canto superior direito da tela para atualizar automaticamente seu aplicativo sempre que você alterar o código-fonte.

Isso permite que você trabalhe em um loop interativo rápido: você digita algum código, salva, experimenta ao vivo, depois digita mais código, salva, experimenta e assim por diante até ficar satisfeito com os resultados. Esse laço estreito entre a codificação e a visualização dos resultados ao vivo é uma das maneiras pelas quais o Streamlit facilita sua vida.

Ao desenvolver um aplicativo Streamlit, é recomendável colocar as janelas do editor e do navegador lado a lado, para que o código e o aplicativo possam ser vistos ao mesmo tempo. De uma chance!

Fluxo de dados

A arquitetura do Streamlit permite que você escreva aplicativos da mesma forma que escreve scripts Python simples. Para desbloquear isso, os aplicativos Streamlit têm um fluxo de dados exclusivo: sempre que algo deve ser atualizado na tela, o Streamlit executa novamente todo o seu script Python de cima para baixo.

Isso pode acontecer em duas situações:

  • Sempre que você modificar o código-fonte do seu aplicativo.

  • Sempre que um usuário interage com widgets no aplicativo. Por exemplo, ao arrastar um controle deslizante, inserir texto em uma caixa de entrada ou clicar em um botão.

Sempre que um retorno de chamada é passado para um widget por meio do parâmetro on_change (ou on_click ), o retorno de chamada sempre será executado antes do resto do seu script. Para obter detalhes sobre a API de retornos de chamada, consulte nosso Guia de referência da API de estado de sessão .

E para tornar tudo isso rápido e contínuo, o Streamlit faz um trabalho pesado para você nos bastidores. Um grande jogador nessa história é o @st.cache decorador, que permite que os desenvolvedores pulem certos cálculos caros quando seus aplicativos forem executados novamente. Abordaremos o cache posteriormente nesta página.

Dados de exibição e estilo

Existem algumas maneiras de exibir dados (tabelas, matrizes, quadros de dados) em aplicativos Streamlit. Abaixo , você será apresentado ao magic e st.write() , que pode ser usado para escrever qualquer coisa, de texto a tabelas. Depois disso, vamos dar uma olhada nos métodos projetados especificamente para visualizar dados.

Usar magia

Você também pode gravar em seu aplicativo sem chamar nenhum método Streamlit. O Streamlit suporta " comandos mágicos ", o que significa que você não precisa usar st.write() nada! Para ver isso em ação, tente este trecho: 

"""
# My first app
Here's our first attempt at using data to create a table:
"""

import streamlit as st
import pandas as pd
df = pd.DataFrame({
  'first column': [1, 2, 3, 4],
  'second column': [10, 20, 30, 40]
})

df

Sempre que o Streamlit vê uma variável ou um valor literal em sua própria linha, ele automaticamente grava isso em seu aplicativo usando st.write() . Para obter mais informações, consulte a documentação sobre comandos mágicos .

Escreva um quadro de dados

Junto com os comandos mágicos , st.write() está o "canivete suíço" da Streamlit. Você pode passar quase qualquer coisa para st.write() : texto, dados, figuras Matplotlib, gráficos Altair e muito mais. Não se preocupe, o Streamlit descobrirá e renderizará as coisas da maneira certa. 

import streamlit as st
import pandas as pd

st.write("Here's our first attempt at using data to create a table:")
st.write(pd.DataFrame({
    'first column': [1, 2, 3, 4],
    'second column': [10, 20, 30, 40]
}))

Existem outras funções específicas de dados como st.dataframe() e st.table() que você também pode usar para exibir dados. Vamos entender quando usar esses recursos e como adicionar cores e estilos aos seus quadros de dados.

Você pode estar se perguntando: "por que eu não usaria sempre st.write() ?" Há algumas razões:

  1. Magic e st.write() inspecione o tipo de dados que você passou e, em seguida, decida como melhor renderizá-los no aplicativo. Às vezes você quer desenhar de outra maneira. Por exemplo, em vez de desenhar um dataframe como uma tabela interativa, você pode querer desenhá-lo como uma tabela estática usando st.table(df) .
  2. A segunda razão é que outros métodos retornam um objeto que pode ser usado e modificado, seja adicionando dados a ele ou substituindo-o.
  3. Por fim, se você usar um método Streamlit mais específico, poderá passar argumentos adicionais para personalizar seu comportamento.

Por exemplo, vamos criar um data frame e alterar sua formatação com um Styler objeto Pandas. Neste exemplo, você usará o Numpy para gerar uma amostra aleatória e o st.dataframe() método para desenhar uma tabela interativa.

O Streamlit também possui um método para geração de tabelas estáticas: st.table() . 

import streamlit as st
import numpy as np
import pandas as pd

dataframe = pd.DataFrame(
    np.random.randn(10, 20),
    columns=('col %d' % i for i in range(20)))
st.table(dataframe)

Desenhe gráficos e mapas

O Streamlit suporta várias bibliotecas de gráficos de dados populares como Matplotlib, Altair, deck.gl e muito mais . Nesta seção, você adicionará um gráfico de barras, um gráfico de linhas e um mapa ao seu aplicativo.

Desenhe um gráfico de linhas

Você pode adicionar facilmente um gráfico de linhas ao seu aplicativo com o st.line_chart() . Vamos gerar uma amostra aleatória usando o Numpy e, em seguida, mapeá-la. 

import streamlit as st
import numpy as np
import pandas as pd

chart_data = pd.DataFrame(
     np.random.randn(20, 3),
     columns=['a', 'b', 'c'])

st.line_chart(chart_data)

Traçar um mapa

Com st.map() você pode exibir pontos de dados em um mapa. Vamos usar o Numpy para gerar alguns dados de amostra e plotá-los em um mapa de São Francisco. 

import streamlit as st
import numpy as np
import pandas as pd

map_data = pd.DataFrame(
    np.random.randn(1000, 2) / [50, 50] + [37.76, -122.4],
    columns=['lat', 'lon'])

st.map(map_data)

Widgets

Quando você tiver os dados ou o modelo no estado que deseja explorar, poderá adicionar widgets como st.slider() , st.button() ou st.selectbox() . É muito simples — trate os widgets como variáveis: 

import streamlit as st
x = st.slider('x')  # 👈 this is a widget
st.write(x, 'squared is', x * x)

Na primeira execução, o aplicativo acima deve gerar o texto "0 ao quadrado é 0". Então, toda vez que um usuário interage com um widget, o Streamlit simplesmente executa novamente seu script de cima para baixo, atribuindo o estado atual do widget à sua variável no processo.

Por exemplo, se o usuário mover o controle deslizante para position 10 , o Streamlit executará novamente o código acima e definirá x de 10 acordo. Então agora você deve ver o texto "10 ao quadrado é 100".

Os widgets também podem ser acessados ​​por chave, se você optar por especificar uma string a ser usada como chave exclusiva para o widget: 

import streamlit as st
st.text_input("Your name", key="name")

# You can access the value at any point with:
st.session_state.name

Cada widget com uma chave é adicionado automaticamente ao estado da sessão. Para obter mais informações sobre o estado da sessão, sua associação com o estado do widget e suas limitações, consulte o Guia de referência da API do estado da sessão .

Use caixas de seleção para mostrar/ocultar dados

Um caso de uso para caixas de seleção é ocultar ou mostrar um gráfico ou seção específica em um aplicativo. st.checkbox() recebe um único argumento, que é o rótulo do widget. Neste exemplo, a caixa de seleção é usada para alternar uma instrução condicional. 

import streamlit as st
import numpy as np
import pandas as pd

if st.checkbox('Show dataframe'):
    chart_data = pd.DataFrame(
       np.random.randn(20, 3),
       columns=['a', 'b', 'c'])

    chart_data

Use uma caixa de seleção para opções

Use st.selectbox para escolher entre uma série. Você pode escrever nas opções desejadas ou passar por uma coluna de matriz ou quadro de dados.

Vamos usar o df quadro de dados que criamos anteriormente. 

import streamlit as st
import pandas as pd

df = pd.DataFrame({
    'first column': [1, 2, 3, 4],
    'second column': [10, 20, 30, 40]
    })

option = st.selectbox(
    'Which number do you like best?',
     df['first column'])

'You selected: ', option

Esquema

O Streamlit facilita a organização de seus widgets em uma barra lateral do painel esquerdo com st.sidebar . Cada elemento para o qual é passado st.sidebar é fixado à esquerda, permitindo que os usuários se concentrem no conteúdo do seu aplicativo enquanto ainda têm acesso aos controles da interface do usuário.

Por exemplo, se você deseja adicionar uma caixa de seleção e um controle deslizante a uma barra lateral, use st.sidebar.slider e st.sidebar.selectbox em vez de st.slider e st.selectbox : 

import streamlit as st

# Add a selectbox to the sidebar:
add_selectbox = st.sidebar.selectbox(
    'How would you like to be contacted?',
    ('Email', 'Home phone', 'Mobile phone')
)

# Add a slider to the sidebar:
add_slider = st.sidebar.slider(
    'Select a range of values',
    0.0, 100.0, (25.0, 75.0)
)

Além da barra lateral, o Streamlit oferece várias outras maneiras de controlar o layout do seu aplicativo. st.columns permite colocar widgets lado a lado e st.expander economizar espaço ocultando conteúdo grande. 

import streamlit as st

left_column, right_column = st.columns(2)
# You can use a column just like st.sidebar:
left_column.button('Press me!')

# Or even better, call Streamlit functions inside a "with" block:
with right_column:
    chosen = st.radio(
        'Sorting hat',
        ("Gryffindor", "Ravenclaw", "Hufflepuff", "Slytherin"))
    st.write(f"You are in {chosen} house!")

Mostrar progresso

Ao adicionar cálculos de longa duração a um aplicativo, você pode usar st.progress() para exibir o status em tempo real.

Primeiro, vamos importar o tempo. Vamos usar o time.sleep() método para simular uma computação de longa duração: 

import time

Agora, vamos criar uma barra de progresso: 

import streamlit as st
import time

'Starting a long computation...'

# Add a placeholder
latest_iteration = st.empty()
bar = st.progress(0)

for i in range(100):
  # Update the progress bar with each iteration.
  latest_iteration.text(f'Iteration {i+1}')
  bar.progress(i + 1)
  time.sleep(0.1)

'...and now we\'re done!'

Temas

O Streamlit suporta temas claros e escuros prontos para uso. O Streamlit verificará primeiro se o usuário que está visualizando um aplicativo tem uma preferência de modo Claro ou Escuro definida por seu sistema operacional e navegador. Em caso afirmativo, essa preferência será usada. Caso contrário, o tema Light é aplicado por padrão.

Você também pode alterar o tema ativo de "☰" → "Configurações".

Deseja adicionar seu próprio tema a um aplicativo? O menu "Configurações" possui um editor de temas acessível clicando em "Editar tema ativo". Você pode usar este editor para experimentar cores diferentes e ver a atualização do seu aplicativo ao vivo.

Quando estiver satisfeito com seu trabalho, os temas podem ser salvos definindo as opções de configuração na [theme] seção de configuração. Depois de definir um tema para seu aplicativo, ele aparecerá como "Tema personalizado" no seletor de temas e será aplicado por padrão em vez dos temas Claro e Escuro incluídos.

Mais informações sobre as opções disponíveis ao definir um tema podem ser encontradas na documentação da opção do tema .

Outra maneira de experimentar diferentes cores de tema é ativar a opção "Executar ao salvar", editar seu arquivo config.toml e observar como seu aplicativo é executado novamente com as novas cores de tema aplicadas.

Cache

prioridade_alta

Importante

Estamos desenvolvendo novas primitivas de cache que são mais fáceis de usar e muito mais rápidas do que @st.cache . 🚀 Para saber mais, leia Primitivos de cache experimental .

O cache Streamlit permite que seu aplicativo seja executado rapidamente, mesmo ao carregar dados da Web, manipular grandes conjuntos de dados ou realizar cálculos caros.

Para usar o cache, envolva funções com o @st.cache decorador: 

import streamlit as st

@st.cache  # 👈 This function will be cached
def my_slow_function(arg1, arg2):
    # Do something really slow in here!
    return the_output

Quando você marca uma função com o @st.cache decorador, ele diz ao Streamlit que sempre que a função é chamada, ele precisa verificar algumas coisas:

  1. Os parâmetros de entrada com os quais você chamou a função
  2. O valor de qualquer variável externa usada na função
  3. O corpo da função
  4. O corpo de qualquer função usada dentro da função em cache

Se esta é a primeira vez que o Streamlit vê esses quatro componentes com esses valores exatos e nessa combinação e ordem exatas, ele executa a função e armazena o resultado em um cache local. Então, na próxima vez que a função em cache for chamada, se nenhum desses componentes for alterado, o Streamlit pulará a execução da função completamente e, em vez disso, retornará a saída armazenada anteriormente no cache.

Para obter mais informações sobre o cache do Streamlit, seus parâmetros de configuração e suas limitações, consulte Cache .

Modelo do aplicativo

Agora que você sabe um pouco mais sobre todas as peças individuais, vamos fechar o ciclo e revisar como elas funcionam juntas:

  1. Aplicativos Streamlit são scripts Python executados de cima para baixo
  2. Sempre que um usuário abre uma guia do navegador apontando para seu aplicativo, o script é executado novamente
  3. À medida que o script é executado, o Streamlit desenha sua saída ao vivo em um navegador
  4. Os scripts usam o cache Streamlit para evitar recalcular funções caras, então as atualizações acontecem muito rápido
  5. Toda vez que um usuário interage com um widget, seu script é executado novamente e o valor de saída desse widget é definido para o novo valor durante essa execução.
